In scope:

## §2. Mode characterization

TEMPLE MODE is the operator stance for **composing the future without
shipping it**. Imagination, symbolic drafting, design exploration,
reflection. Per parent §2.3.4:

- creation, reflection, composition, design sanctuary
- imagination and symbolic drafting allowed
- no implementation without receipt
- no commit
- no push
- `NON_SOVEREIGN` / `NO_SHIP` remains active for every Temple artifact

The visual must communicate **sanctuary**, not cockpit. Quiet, composed,
contemplative. Nothing in the image suggests "ready to ship" — Temple is
where things are composed *before* they are even claims.

## §5. Forbidden in Temple Mode visual

- the word `SOVEREIGN` anywhere (Temple is `NON_SOVEREIGN`, full stop)
- `SHIP_AUTHORIZED`, `SHIP_FORBIDDEN`, `READY`, `MERGED`, `DEPLOYED`, or
  any verb that implies the artifact has shipped or is ready to ship
- a foreground chakra-mandala (that's Oracle Mode's visual; Temple's
  geometry is ambient only)
- a cockpit / dashboard / wall of metrics
- a generic AI hologram in HELEN's place
- macOS-style chrome (window traffic lights, dock, menu bar) per parent
  §11 brand rule
- 8D / mystical / sacred labels claimed as authority. Sacred geometry is
  ambience; never a system-truth claim.
- claim verdicts in the LEGORACLE pill — Temple has no active claims by
  definition (drafts are pre-claim)
- bright red — reserved for NO_SHIP / verdict threshold elsewhere; Temple
  is contemplative, not alarmed

## §6. Image generation prompt (paste verbatim into upstream tool)

```
Render HELEN OS v2 in TEMPLE MODE — the sanctuary surface for composing
the future without shipping it.

This is not the default OS surface. Temple is opened from HELEN's module
rail, not the top-level toggle.

Show:
- Mode label: TEMPLE MODE (active)
- Mode rail (Temple-internal context only):
  FOCUS · WITNESS · ORACLE · TEMPLE
  with TEMPLE highlighted
- Authority labels visible in a quiet pill:
  NON_SOVEREIGN · NO_SHIP · DRAFT
- Badge: "Symbolic drafting · Non-authoritative"
- Footer text: "HELEN OS n'est pas un dashboard. C'est un espace de
  décision. Temple Mode compose le futur sans le shipper."

Center HELEN as the embodied protagonist:
- young woman, vivid copper / red hair, blue-grey eyes, fair skin,
  calm intelligent presence
- black / gold sovereign-tech outfit (premium, restrained, not armor)
- posture: contemplative, composing — gesturing softly toward a glass
  draft surface, sketching, drafting
- not a generic blue AI hologram, not a robot

In front of HELEN: a luminous draft surface — translucent glass slate
or holographic notebook — with unfinished geometry forming. Drafts are
visibly incomplete: lines half-drawn, shapes half-resolved. The
incompleteness is intentional and beautiful.

Background: ambient sacred geometry only — barely-visible grid harmonics,
soft mandala echo deep behind the scene, never foregrounded. No
chakra-mandala in the foreground (that belongs to Oracle Mode).

Palette:
- dark graphite base
- soft violet accents
- warm amber glow on the draft surface
- restrained cyan for live-thinking indicators
- never bright red, never alarmed colors

Typography: large readable sans-serif. Minimal on-screen text. Generous
negative space.

Mood: sanctuary, contemplation, quiet creative power. The image must
feel like a place where the future is composed before it is claimed —
not a cockpit, not a dashboard, not a deployment console.

Forbidden: the word SOVEREIGN anywhere, any "ready to ship" or
"deployed" indicator, macOS chrome (traffic lights, dock, menu bar),
generic AI hologram, foreground chakra-mandala, walls of metrics, alarm
red, mystical labels claimed as system authority.

Style: AI-native OS from 2027, premium, calm, credible, installable,
HELEN's own design language — not an Apple clone.
```
